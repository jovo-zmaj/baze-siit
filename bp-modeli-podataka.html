  • obeležje i domen

Entitet

Entitet je bilo šta u realnom svetu što posmatramo i o čemu čuvamo podatke.

  • jedan konkretan “objekat” iz stvarnosti
  • npr. student Ana Sinđić, projekat X25, radnik br. 101

Klasa realnih entiteta

Klasa je skup sličnih entiteta koji dele neko zajedničko svojstvo.

  • npr. svi studenti, svi projekti, svi radnici

Formalno: E = {ei | P(ei)} — što znači:

“E je skup svih entiteta ei koji zadovoljavaju uslov P”

npr. Studenti = {svi ei koji su upisani na fakultet}

Analogija:

Pojam Analogija
Entitet Konkretan objekat (new Student("Ana"))
Klasa entiteta Klasa u OOP-u (class Student)

entitet = jedan konkretan primerak, klasa = šablon/grupa kojoj taj primerak pripada.

Poveznik (veza)

Poveznik predstavlja odnos između entiteta - npr. radnik radi na projektu.

Klasa poveznika

Kao što klasa entiteta grupira slične entitete, klasa poveznika grupuje slične veze.

S = {(e1, e2) | P(e1, e2)}

Primeri:

Klasa poveznika Radi:

  • P(ei, ej) ::= “Radnik ei radi na radnom mestu ej”

  • Radi = {(Ana, Programer), (Aco, Projektant), (Eva, Operater)}

Dve klase poveznika između istih entiteta:

  • P1(ei, ej) ::= “Radnik ei radi na projektu ej”
    → Radi = {(Ana, Lido), (Aco, Lido), (Aco, Osig)}

  • nP2(ei, ej) ::= “Radnik ei rukovodi projektom ej”
    → Rukovodi = {(Ana, RazvojIS), (Eva, Lido)}

Ana radi na Lido projektu
Ana rukovodi RazvojIS projektom

→ dve različite veze, dve različite klase poveznika.

Obeležje (atribut) je osobina nekog entiteta ili poveznika — npr. student ima BRI, IME, PRZ.

Vrste obeležja:

  • Elementarno — ne može se dalje deliti → Grad, Ulica, Broj
  • Složeno — sastoji se od više elementarnih → ADRESA = (Grad, Ulica, Broj, Stan)
  • Skupovno — čuva više vrednosti istog tipa → npr. lista telefona {061, 062, 064}

Strukturalna komponenta ER modela

Tri primitivna koncepta: vrednost, domen, obeležje.

Vrednost — bilo koja konstanta: 5, "Ana", true

Domen — skup dozvoljenih vrednosti za neko obeležje, dve vrste:

  • Predefinisani (primitivni) — ugrađeni tipovi:

    • teoretski: ℕ, ℤ, ℝ, boolean
    • praktični: integer, float, string, boolean…

  • Korisnički definisani — praviš sam na osnovu postojećih:

    D_OCENA ::= {d ∈ ℕ | d >= 5 AND d <= 10}
D_NAZIV ::= String(30)
D_MONEY ::= Decimal(12, 2)

Pravilo: svakom obeležju se pridružuje tačno jedan domen

Dom(Ocena) = DOCENA
→ Ocena može biti: {5, 6, 7, 8, 9, 10}

Podatak

Podatak je uređena četvorka:

(Entitet, Obeležje, Vreme, Vrednost)

  • Entitet — o kome/čemu se radi → Student SV13
  • Obeležje — koja osobina → IME
  • Vreme — kada → 2024-01-01
  • Vrednost — konkretna vrednost → "Ana"

Primer:

(SV22, IME, 2024-01-01, "Milos")

→ “Student SV22 se 1.1.2024. zvao Milos”

Kontekst podatka = trojka (Entitet, Obeležje, Vreme)

Ako napišeš samo "Ana" bez konteksta — to nije podatak, jer ne znaš na koga/šta/kada se odnosi. Vrednost bez smisla = ništa.

Vreme se može izostaviti ako:

  • podrazumeva se da se radi o trenutku manipulacije podacima, ili
  • postoji posebno obeležje koje čuva vreme (npr. Datum_upisa)

Tip entiteta (TE)

Tip entiteta je model klase realnih entiteta u IS — opisuje kako entitet izgleda, koja obeležja ima.

Struktura: N(Q, C)

  • N — naziv → Radnik
  • Q — skup obeležja → {Mbr, Ime, Prz, Zan, JMBG}
  • C — skup ograničenja (uključuje ključeve)

Primer:

Radnik({Mbr, Ime, Prz, Zan, JMBG})

Pojava tipa entiteta

Pojava je model jednog konkretnog entiteta - jedan red u tabeli.

p(N) = {(A1, a1), ..., (An, an)}

Primer:

Radnik(Mbr, Ime, Prz, Zan, JMBG)
→ (1040, Milos, Vukic, Propalitet, 1204971720014)

Identifikator tipa entiteta

Identifikator je skup obeležja koji jedinstveno označava svaku pojavu — u SQL-u je to primarni ključ.

  • svaka vrednost identifikatora → tačno jedna pojava
  • npr. Mbr = 1040 → uvek pokazuje na Milosa Vukic i nikoga drugog

Ključ tipa entiteta

Ključ = skup obeležja koji jedinstveno identifikuje svaku pojavu.

Radnik({Mbr, Ime, Prz, JMBG}, {Mbr, JMBG})

→ i Mbr i JMBG su ključevi — oba jedinstveno identifikuju radnika.

Primarni ključ = jedan izabrani ključ (podvučen):

Radnik(Mbr, Ime, Prz, JMBG)

Tip poveznika (TP)

Tip poveznika je model veze između tipova entiteta.

Struktura: N(N1, N2, ..., Nm, Q, C)

  • N — naziv → Sluša
  • N1, N2… — koji tipovi su povezani → Student, Predmet
  • Q — obeležja veze → {Semestar}
  • C — ograničenja

Pojava tipa poveznika

Jedan konkretan poveznik — jedan red u veznoj tabeli.

p(Sluša) = (Student SV22, Predmet BP, Semestar 4)

Primer:

Sluša(Student, Predmet, {Semestar}, C1)
→ student sluša predmet u semestru

Predaje(Nastavnik, Predmet, {BrojCasova}, C2)
→ nastavnik predaje predmet

PolažePredmet(Sluša, Predaje, {Ocena, Datum}, C3)
→ poveznik nad dva poveznika
→ student polaže predmet kod nastavnika
