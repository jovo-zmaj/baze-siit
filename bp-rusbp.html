Realizacija ograničenja pomoću RSUBP – Skripta za učenje

Šta je cilj?

RSUBP omogućava da u bazi definišemo pravila (ograničenja) kako bi
podaci bili ispravni.

Najčešći načini: - CREATE DOMAIN - CONSTRAINT (NOT NULL, CHECK,
PRIMARY KEY, FOREIGN KEY) - CREATE TRIGGER

1. Ograničenje domena

Određuje kakve vrednosti sme da ima kolona.

Može da se realizuje pomoću: - CREATE DOMAIN - CHECK - TRIGGER

Primer:

CHECK (Plt > 500)

➡ Plata mora biti veća od 500.

2. Ograničenje vrednosti obeležja

Odnosi se na jednu kolonu.

Najčešće:

Ime VARCHAR(20) NOT NULL

NOT NULL znači da kolona ne sme biti prazna.

3. Ograničenje torke

Odnosi se na više kolona istovremeno.

Primer:

CHECK (SUBSTR(JMBG,1,7)=TO_CHAR(DATR,'DDMMYYY'))

Značenje: - prvih 7 cifara JMBG mora da odgovara datumu rođenja.

Za ovakva pravila koristi se CHECK ili TRIGGER.

Primer tabele

CREATE TABLE Radnik(
    Mbr INTEGER NOT NULL,
    Ime VARCHAR(20) NOT NULL,
    Prz VARCHAR(25) NOT NULL,
    Sef INTEGER,
    Plt DECIMAL(10,2),
    Pre DECIMAL(6,2),
    God DATE NOT NULL,

    CONSTRAINT radnik_PK PRIMARY KEY(Mbr),
    CONSTRAINT radnik_FK FOREIGN KEY(Sef)
        REFERENCES Radnik(Mbr),
    CONSTRAINT radnik_CH CHECK(Plt > 500)
);

Šta rade ograničenja?

  • NOT NULL → vrednost mora postojati.
  • PRIMARY KEY → jedinstveno identifikuje svaki red.
  • FOREIGN KEY → povezuje tabelu sa drugom tabelom.
  • CHECK → proverava uslov.

Kada koristiti koji mehanizam?

Situacija Koristi se

Ograničenje jedne kolone NOT NULL, CHECK
Ograničenje više kolona CHECK
Povezivanje tabela FOREIGN KEY
Složena pravila TRIGGER
Definisanje domena CREATE DOMAIN

Šta treba zapamtiti za ispit

  • CREATE DOMAIN definiše dozvoljene vrednosti domena.
  • NOT NULL zabranjuje NULL vrednosti.
  • CHECK proverava logički uslov.
  • PRIMARY KEY = jedinstven + ne sme biti NULL.
  • FOREIGN KEY povezuje dve tabele.
  • TRIGGER se koristi kada CHECK nije dovoljan ili je potrebno
    izvršiti dodatne akcije.
